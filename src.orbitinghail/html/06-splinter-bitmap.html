Orbitinghail -- Splinter-RS Compressed Bitmap

Splinter-rs is a compressed bitmap for small, sparse sets of u32s with zero-copy querying. It is used by graft for PageSet tracking and by lsm-tree for block index compaction. The bitmap uses a combination of run-length encoding and sparse representation to achieve high compression for sparse sets.

Aha: Splinter-rs uses u24 (24-bit integers) for compact storage within the bitmap structure. Since page indices and block offsets rarely exceed 16 million (2^24), using u24 instead of u32 saves 25% of space. Combined with zerocopy for safe transmutation between u24 and byte slices, the bitmap can be read and written without deserialization — true zero-copy.

Bitmap Structure

pub struct Bitmap {
    data: Vec<u8>,           // Compressed bitmap data
    cardinality: u32,        // Number of set bits
    checksum: u64,           // CRC64-NVME for integrity
}

Operations

OperationDescriptionComplexity
`insert(u32)`Set a bitO(n) where n = bitmap size
`contains(u32) -> bool`Check if bit is setO(log n) with binary search
`cardinality() -> u32`Count of set bitsO(1) — cached
`union(&Bitmap) -> Bitmap`Set unionO(n + m)
`intersection(&Bitmap) -> Bitmap`Set intersectionO(min(n, m))
`cut(&Range) -> Bitmap`Remove bits in rangeO(n)
`select(n: u32) -> Option`Find n-th set bitO(n)
`iter() -> impl Iterator`Iterate over set bitsO(cardinality)

Compression Strategy

The bitmap uses different encoding strategies based on density:

flowchart TD A["New bitmap"] --> B{"Density > threshold?"} B -->|Sparse| C["Sorted array of set values"] B -->|Dense| D["Bit vector with run-length encoding"] C --> E["CRC64 checksum"] D --> E

For sparse sets (<10% density): Store sorted array of set values. Each value is a u24 (3 bytes) instead of a full u32.

For dense sets (>10% density): Use a bit vector with run-length encoding. Consecutive runs of 0s or 1s are compressed as (value, length) pairs.

CRC64-NVME Integrity

use crc64fast_nvme::Digest;

let mut digest = Digest::new();
digest.update(&self.data);
self.checksum = digest.finalize();

CRC64-NVME is a fast 64-bit CRC designed for storage integrity. It is computed over the compressed bitmap data and stored alongside it. On read, the checksum is verified — if it doesn't match, the bitmap is corrupted.

u24 Integer Type

Source: u24/src/lib.rs

#[repr(align(4))]
pub struct u24 {
    data: [u8; 3],  // 3 bytes
    _padding: u8,   // 1 byte padding for 4-byte alignment
}

The u24 type stores 3 bytes of data with 1 byte of padding to match u32 alignment. This allows:

  • Zero-copy transmutation via zerocopy
  • 4-byte alignment for efficient memory access
  • No endianness issues (big-endian encoding)
use zerocopy::{FromBytes, IntoBytes};

let val = u24::from_u32(1_000_000).unwrap();
let bytes: [u8; 4] = val.as_bytes();  // [0, 0x0F, 0x42, 0x40]
let restored = u24::ref_from_bytes(&bytes).unwrap();

Usage in Graft

In graft, PageSet tracks which pages are in a volume:

// Pages 1, 5, 100, 1000 set in a 10,000-page volume
let mut pageset = Bitmap::new();
pageset.insert(1);
pageset.insert(5);
pageset.insert(100);
pageset.insert(1000);

// Check if page 5 is in the set
assert!(pageset.contains(5));
assert!(!pageset.contains(6));

// Union with another pageset
let pageset2 = ...;
let combined = pageset.union(&pageset2);

For a volume with 1 million pages where only 1000 are set, the sparse array uses 1000 × 3 = 3000 bytes instead of 1 million bits = 125,000 bytes.

Usage in LSM-Tree

In lsm-tree, splinter-rs is used for block index tracking and prefix compression metadata. The compressed bitmap reduces the size of the binary index for large SSTables.

Replicating in Rust

use splinter_rs::Bitmap;

// Create and populate
let mut bm = Bitmap::new();
for i in [1, 5, 100, 1000] {
    bm.insert(i);
}

// Query
assert_eq!(bm.cardinality(), 4);
assert!(bm.contains(100));

// Set operations
let other = Bitmap::from_iter([5, 200, 300]);
let union = bm.union(&other);     // {1, 5, 100, 200, 300, 1000}
let intersection = bm.intersection(&other);  // {5}

// Iterate
for value in bm.iter() {
    println!("Set bit: {}", value);
}

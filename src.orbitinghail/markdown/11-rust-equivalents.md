# Orbitinghail -- Rust Equivalents and Patterns

This document maps every orbitinghail concept to idiomatic Rust patterns, showing how to use these crates in your own projects and what production-grade alternatives exist.

## LSM-Tree Usage

### Basic KV Store

```rust
use lsm_tree::{Config, Tree};

let config = Config::new("/tmp/my-store", seqno, visible_seqno)
    .write_buffer_size_policy(16 * 1_024 * 1_024)
    .data_block_size_policy(4096)
    .build()?;

let tree = config.open()?;
tree.insert(b"key", b"value")?;
tree.insert(b"key2", b"value2")?;

let value = tree.get(b"key")?;
assert_eq!(value.as_deref(), Some(b"value".as_slice()));

// Iterate over a range
for entry in tree.scan(b"a"..=b"z")? {
    println!("{:?} -> {:?}", entry.key(), entry.value());
}
```

### With Bloom Filter

```rust
let config = Config::new("/tmp/my-store", seqno, visible_seqno)
    .filter_policy(FilterPolicy::BitsPerKey(10.0))
    .build()?;
```

**Production pattern:** For production workloads, configure the bloom filter's false positive rate based on your read/write ratio. A write-heavy workload benefits from a lower false positive rate (fewer unnecessary SSTable reads). A read-heavy workload can tolerate a higher rate.

## Fjall Database

### Multi-Keyspace with Atomic Batch

```rust
use fjall::{ConfigBuilder, PersistMode};

let db = ConfigBuilder::new()
    .path("/tmp/my-db")
    .open()?;

let users = db.open_keyspace("users")?;
let sessions = db.open_keyspace("sessions")?;

// Atomic batch across keyspaces via single_writer_tx
let tx = db.single_writer_tx()?;
tx.insert(&users, b"user:1", b"Alice")?;
tx.insert(&sessions, b"sess:abc", b"user:1")?;
tx.commit()?;

db.persist(PersistMode::SyncAll)?;
```

### Transactional Update

```rust
let tx = db.optimistic_tx()?;
let old_value = tx.get(&users, b"counter")?;
let new_value = parse_counter(&old_value)? + 1;
tx.insert(&users, b"counter", new_value.to_string().into_bytes())?;

match tx.commit() {
    Ok(()) => println!("Committed"),
    Err(fjall::CommitError::Conflict) => println!("Retry"),
    Err(e) => return Err(e),
}
```

**Production pattern:** Optimistic transactions work well for low-contention workloads. For high-contention (many threads updating the same key), use `SingleWriterTxDatabase` with external serialization or switch to pessimistic locking.

## Graft Volume

```rust
use graft::{Runtime, GraftConfig, RemoteConfig, Volume, PageIdx, FjallStorage};

// fjall database
let db = ConfigBuilder::new().path("/tmp/my-db").open()?;
let storage = FjallStorage::new(&db)?;

// remote config
let remote = RemoteConfig::S3Compatible {
    bucket: "my-bucket".to_string(),
    region: "us-east-1".to_string(),
    // credentials
};

// autosync configured via GraftConfig
let config = GraftConfig::new(storage, remote).autosync(true);
let runtime = Runtime::new(tokio_handle, remote, storage, config)?;

// Open volume
let volume = runtime.volume("my-data")?;

// Write page
volume.write_page(PageIdx::new(1)?, page_data)?;

// Read page
let page = volume.read_page(PageIdx::new(1)?)?;

// Background sync runs automatically when autosync is enabled
```

**Production pattern:** For offline-first applications, open the volume immediately with local-only mode and start the autosync task when connectivity is detected.

## Splinter-RS Bitmap

```rust
use splinter_rs::Splinter;

// Sparse set
let mut bm = Splinter::new();
for i in [1, 5, 100, 1000, 5000] {
    bm.insert(i);
}

assert!(bm.contains(100));
assert!(!bm.contains(101));
assert_eq!(bm.cardinality(), 5);

// Set operations
let other = Splinter::from_iter([5, 200, 300]);
let union = bm.union(&other);
let intersection = bm.intersection(&other);
```

## SQLSync WASM

```typescript
// TypeScript (generated by tsify)
import { Database } from './sqlsync_wasm';

const db = await Database.open('my-volume');
const conn = await db.connect();

const rows = await conn.query(
  'SELECT * FROM users WHERE id = ?',
  [1]
);

// Replication
const frames = await conn.getFrameRange(1n, 100n);
await sendToServer(frames);
```

SSH Integration

SSH login with rauthy.

Overview

Two SSH authentication methods:

  1. PAM Remote Password — Password via rauthy
  2. Public Keys — Uploaded to rauthy

PAM Remote Password

Flow

sequenceDiagram participant User participant Client participant Server participant PAM participant Rauthy User->>Rauthy: Login to dashboard User->>Rauthy: Generate PAM Remote Password Rauthy-->>User: Temporary password User->>Client: ssh user@server Client->>Server: Connect Server->>User: Password prompt User->>Server: Enter PAM Remote Password Server->>PAM: pam_authenticate() PAM->>Rauthy: Validate Rauthy-->>PAM: Success PAM-->>Server: Auth success Server-->>Client: Logged in

Configuration

# /etc/pam.d/sshd
auth    sufficient    pam_rauthy.so
auth    required      pam_unix.so

Public Key Authentication

AuthorizedKeysCommand

# /etc/ssh/sshd_config
AuthorizedKeysCommand /usr/bin/rauthy-authorized-keys %u
AuthorizedKeysCommandUser root

Implementation

// src/rauthy-authorized-keys/main.rs
fn main() {
    let username = env::args().nth(1).expect("username required");
    
    // Fetch keys from rauthy
    let keys = rauthy_client::get_user_ssh_keys(&username)
        .expect("fetch keys");
    
    // Print authorized_keys format
    for key in keys {
        println!("{} {}", key.algorithm, key.key);
    }
}

SSH Config

Server Configuration

# /etc/ssh/sshd_config
# Authentication
PasswordAuthentication yes
PubkeyAuthentication yes

# PAM
UsePAM yes

# Authorized keys command
AuthorizedKeysCommand /usr/bin/rauthy-authorized-keys %u
AuthorizedKeysCommandUser root

# Allow rauthy users
AllowUsers *@rauthy

sudo Access

# /etc/sudoers
%wheel-rauthy   ALL=(ALL)   ALL

Rauthy groups can be mapped to local sudo access.

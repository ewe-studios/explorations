# Zero to Joy Developer

## Overview

Joy is a collection of Go-based tools for modern web development. This guide covers the main projects in the Joy ecosystem:

1. **Joy Compiler** - Translates idiomatic Go into concise JavaScript
2. **Bud Framework** - Full-stack web framework for Go
3. **X-ray** - Web scraping library with pagination and crawler support
4. **LLM** - Agent framework for building AI agents across providers

Note: Joy compiler development is currently on hold in favor of Go->WASM development, but the codebase contains valuable patterns for Go-to-JavaScript compilation.

## Prerequisites

- Go 1.17+ (Go 1.18+ recommended for generics)
- Node.js 16+
- C++ compiler (for cgo/V8 compilation)

## Part 1: Joy Compiler

### What is Joy Compiler?

Joy Compiler translates idiomatic Go into concise JavaScript that works in every browser. It uses Go's type system and tooling to build large web applications.

### How Joy Works

Joy follows a multi-stage compilation process:

```
Go Source → Parse → Index → Graph → Translate → Assemble → JavaScript
```

#### Stage 1: Parse

The parser loads Go packages and builds a dependency graph:

```go
// Parse loads, indexes and graphs the program
func Parse(cfg *Config) (idx *index.Index, g *graph.Graph, err error) {
    // Load Go packages
    program, err := loader.Load(&loader.Config{
        JoyPath:  cfg.JoyPath,
        Packages: cfg.Packages,
    })
    
    // Build index of all definitions
    idx, err = indexer.New(program)
    
    // Build dependency graph starting from main()
    graph = buildDependencyGraph(idx)
    
    return idx, graph, nil
}
```

#### Stage 2: Index

The indexer catalogs all Go definitions:

```go
// Definitions tracked by the indexer
type Definition interface {
    ID() string
    Path() string
    Kind() string  // "function", "struct", "method", "interface", etc.
    Name() string
    Exported() bool
    Omitted() bool
    Dependencies() ([]Definition, error)
    Imports() map[string]string
}
```

#### Stage 3: Graph

Builds a topological sort of dependencies:

```go
// Build dependency graph
func buildDependencyGraph(idx *index.Index) *graph.Graph {
    g := graph.New()
    mains := idx.Mains()
    
    // BFS to find all dependencies
    queue := append([]Definition{}, mains...)
    visited := map[string]bool{}
    
    for len(queue) > 0 {
        def := queue[0]
        queue = queue[1:]
        
        if visited[def.ID()] {
            continue
        }
        visited[def.ID()] = true
        
        deps, _ := def.Dependencies()
        for _, dep := range deps {
            g.Edge(def, dep)  // Add edge def -> dep
            if !visited[dep.ID()] {
                queue = append(queue, dep)
            }
        }
    }
    
    return g
}
```

#### Stage 4: Translate

Translates Go AST to JavaScript AST:

```go
// Go function
func add(a, b int) int {
    return a + b
}

// Translates to JavaScript AST
function add(a, b) {
    return a + b;
}
```

The translator handles:
- Functions and methods
- Structs to JavaScript objects
- Interfaces to duck-typed contracts
- Go standard library mappings to JavaScript equivalents

#### Stage 5: Assemble

Generates JavaScript code from AST:

```go
// Module pattern generated by Joy
var pkg = {};

pkg["main"] = (function() {
    var fmt = pkg["github.com/matthewmueller/fmt"];
    
    function add(a, b) {
        return a + b;
    }
    
    return { add: add };
})();

pkg["main"].main();
```

### Using the Joy Compiler

```bash
# Install Joy
curl -sf https://raw.githubusercontent.com/matthewmueller/joy/master/install.sh | sh

# Compile a Go file to JavaScript
joy main.go

# Run in headless Chrome
joy run main.go

# Development server with live reload
joy serve main.go

# Build for development
joy build --dev main.go
```

### Example: Hello World

```go
// main.go
package main

import "github.com/matthewmueller/joy/dom"

func main() {
    el := dom.GetElementByID("app")
    el.SetInnerHTML("Hello, World!")
}
```

Compiles to:

```javascript
(function() {
    var pkg = {};
    
    pkg["main"] = (function() {
        var dom = pkg["github.com/matthewmueller/joy/dom"];
        
        function main() {
            var el = dom.GetElementByID("app");
            el.SetInnerHTML("Hello, World!");
        }
        
        return { main: main };
    })();
    
    pkg["main"].main();
})();
```

## Part 2: Bud Framework

### What is Bud?

Bud is a full-stack web framework for Go that generates boring code automatically. It feels like using a modern JavaScript framework but with Go's type safety.

### Project Structure

```
myapp/
├── app/
│   ├── controller/
│   │   └── hello/
│   │       └── hello.go
│   ├── view/
│   │   └── hello/
│   │       └── index.svelte
│   └── public/
│   │       └── favicon.ico
├── bud/                  # Generated code (can be gitignored)
├── go.mod
└── main.go
```

### Creating a New Project

```bash
# Create new project
bud create hello
cd hello

# Run development server
bud run
```

### Controllers

```go
// app/controller/hello/hello.go
package hello

import (
    "github.com/livebud/bud/framework/controller"
)

type Controller struct{}

// GET /hello
func (c *Controller) Get() string {
    return "Hello, World!"
}

// GET /hello/:name
func (c *Controller) GetShow(name string) string {
    return "Hello, " + name + "!"
}

// POST /hello
func (c *Controller) Post(name string) string {
    return "Created: " + name
}
```

### Views with Svelte

```svelte
<!-- app/view/hello/index.svelte -->
<script>
    export let name = "World"
</script>

<h1>Hello, {name}!</h1>
```

### Bud's Code Generation

Bud generates code on-demand:

1. **Controllers** → HTTP handlers
2. **Views** → SSR and client-side bundles
3. **Public files** → Embedded assets
4. **Transforms** → CSS preprocessing, minification

Generated code lives in `bud/` and can be regenerated anytime.

### Build System

```bash
# Development build
bud build --dev

# Production build
bud build

# The production binary contains everything
./bud/build/myapp
```

## Part 3: X-ray Web Scraper

### What is X-ray?

X-ray is a web scraping library with flexible schema, pagination support, and crawler capabilities.

### Installation

```bash
npm install x-ray
```

### Basic Usage

```javascript
var Xray = require('x-ray')
var x = Xray()

// Scrape a single value
x('https://blog.ycombinator.com/', '.post', [
  {
    title: 'h1 a',
    link: '.article-title@href'
  }
])
  .paginate('.nav-previous a@href')
  .limit(3)
  .write('results.json')
```

### Selector API

```javascript
// Select text content
x('http://google.com', 'title')(fn)

// Select by class
x('http://reddit.com', '.content')(fn)

// Select attribute
x('http://techcrunch.com', 'img.logo@src')(fn)

// Select innerHTML
x('http://news.ycombinator.com', 'body@html')(fn)
```

### Collections

```javascript
// Array of objects
x('https://dribbble.com', 'li.group', [
  {
    title: '.dribbble-img strong',
    image: '.dribbble-img [data-src]@data-src'
  }
])(fn)

// Nested collections
x('http://mat.io', {
  title: 'title',
  items: x('.item', [
    {
      title: '.item-content h2',
      description: '.item-content section'
    }
  ])
})(fn)
```

### Pagination

```javascript
x('https://example.com', '.item', ['title'])
  .paginate('.next a@href')  // Follow next link
  .limit(10)                   // Max 10 pages
  .delay('1s', '3s')          // Random delay between requests
  .concurrency(2)             // 2 concurrent requests
  .then(function(results) {
    console.log(results)
  })
```

### Responsible Scraping

```javascript
var x = Xray()
  .delay('1s', '5s')      // Delay between requests
  .concurrency(3)         // Max 3 concurrent
  .throttle(10, '1s')     // Max 10 requests per second
  .timeout(30000)         // 30 second timeout
```

## Part 4: LLM Agent Framework

### What is LLM?

LLM is a Go library for building AI agents across multiple providers (OpenAI, Anthropic, Gemini, Ollama).

### Installation

```bash
# Install CLI
go install github.com/matthewmueller/llm/cmd/llm@latest

# Install library
go get github.com/matthewmueller/llm
```

### Basic Usage

```go
package main

import (
    "context"
    "fmt"
    "os"
    
    "github.com/matthewmueller/llm"
    "github.com/matthewmueller/llm/providers/openai"
    "github.com/matthewmueller/llm/providers/anthropic"
)

func main() {
    ctx := context.Background()
    
    client := llm.New(
        openai.New(os.Getenv("OPENAI_API_KEY")),
        anthropic.New(os.Getenv("ANTHROPIC_API_KEY")),
    )
    
    // Define a tool
    add := llm.Func("add", "Add two numbers", func(ctx context.Context, in struct {
        A int `json:"a" description:"First number"`
        B int `json:"b" description:"Second number"`
    }) (int, error) {
        return in.A + in.B, nil
    })
    
    // Chat with tool use
    for event, err := range client.Chat(
        ctx,
        "openai",
        llm.WithModel("gpt-4"),
        llm.WithMessage(llm.UserMessage("Add 20 and 22")),
        llm.WithTool(add),
    ) {
        if err != nil {
            panic(err)
        }
        fmt.Print(event.Content)
    }
}
```

### Features

- **Multiple Providers**: OpenAI, Anthropic, Gemini, Ollama
- **Streaming**: Real-time response streaming
- **Tool Calling**: Recursive, concurrent tool execution
- **Thinking Controls**: none, low, medium, high
- **Sandboxing**: Container and local execution

### CLI Usage

```bash
# Set provider
export LLM_PROVIDER=openai
export OPENAI_API_KEY=sk-...

# List models
llm models

# One-shot prompt
llm "Explain CAP theorem in 3 bullets"

# Interactive REPL
llm

# With thinking level
llm -t low "Plan a weekend trip"
```

## Summary

The Joy ecosystem provides:

1. **Joy Compiler** - Go to JavaScript translation (paused, but valuable patterns)
2. **Bud** - Full-stack Go web framework with code generation
3. **X-ray** - Flexible web scraping with pagination
4. **LLM** - Cross-provider AI agent framework

Each project demonstrates different aspects of Go-based tooling for web development.
